Capabilities

Everything you need for
SIF PFD analysis

Built on the PyPFD library by Rafael Rocha, running entirely in the browser via Pyodide. No installation, no backend, no data leaves your machine.

🔢

Triple Independent Calculations

Three dedicated panels for Sensor, Logic Solver and Final Element. Each computes PFDavg independently using both RBD and IEC 61508-N2595 methods.

🏗️

MooN Architecture Support

Configure M-out-of-N voting for any combination from 1oo1 to 9oo9. Supports 1oo1, 1oo2, 2oo2, 2oo3 and more, with automatic validation of M ≤ N.

🔬

Partial Stroke Testing

Enable partial test mode per subsystem. Configure the manual test interval (MT) and partial test coverage (PTC) to model real-world proof test strategies.

📚

XML Component Library

Load a shared XML library of pre-configured components over HTTP. Dropdowns auto-filter by subsystem type — Sensor, Logic, Final Element. Populate all fields with one click.

📊

System PFD & SIL Classification

Series combination of all three subsystems gives the total system PFDavg. Automatic SIL classification (SIL 1–4) with a visual contribution bar chart.

🖨️

PDF & JSON Export

Export the complete A4 landscape report as a high-resolution PDF (3× scale via html2canvas + jsPDF). Save and reload project state as JSON for later use.

🏭

Component Identification

Each calculation stores Name, Manufacturer, Model and Description — filling the gap between raw failure rate data and real equipment traceability.

⚙️

CCF Factor Inputs

Configure independent β and βd Common Cause Failure factors per subsystem to account for dependent failures in redundant architectures.

🖥️

Zero-Dependency Browser App

Python engine loaded via Pyodide (WebAssembly). No server, no build step, no framework. Open the HTML file — it works. Deployable to any static host or GitHub Pages.

Workflow

How to use it

From opening the file to an exported PDF report in a few steps.

1

Open the HTML file

Open PFD_Triple_Calculator.html in any modern browser. Pyodide will automatically download and install the PyPFD engine in the background.

2

Load the component library (optional)

Paste the URL of a pfd_library.xml file into the toolbar and click Load. The dropdowns in each panel will populate with pre-configured components.

3

Fill in each subsystem

Enter Name, Manufacturer, Model and Description for identification. Input failure rates (λdu, λdd), test interval, MTTR, CCF factors and MooN architecture — or select from the library dropdown.

4

Calculate PFD per subsystem

Click ▶ Calculate PFD in each panel. The Python engine runs the RBD and N2595 algorithms and displays PFDavg for that subsystem.

5

Review the SIF Summary

The bottom bar shows all three PFDavg values, the series combination system PFDavg, the resulting SIL classification, and the proportional contribution bar chart.

6

Export

Click 🖨 Export PDF to download a high-resolution A4 landscape PDF. Use 💾 Save JSON to save the project state and reload it later.

Standards compliance

Calculations follow the simplified analytical formulas from IEC 61508 and the tabular method from ISA-TR84.00.02 (N2595), implemented in the open-source PyPFD library.


Both PFDavg values (RBD and N2595) are shown for cross-verification. The system PFD uses series combination: PFDsys = Σ PFDi.

Component Library

XML library format

Host a pfd_library.xml file anywhere accessible over HTTP — GitHub raw files, GitHub Pages, or any static server. The calculator fetches and parses it at runtime.

pfd_library.xml 
<?xml version="1.0" encoding="UTF-8"?>
<pfd_library version="2.0" standard="IEC 61508 / ISA-84">

  <category type="sensor">          <!-- sensor | logic | final_element -->
    <component id="S01">
      <name>Pressure Transmitter (1oo2)</name>
      <manufacturer>Emerson</manufacturer>
      <model>Rosemount 3051S</model>
      <description>Redundant PT, 1oo2 voting</description>
      <ldu>1.0e-6</ldu>            <!-- dangerous undetected rate (h⁻¹) -->
      <ldd>2.5e-5</ldd>            <!-- dangerous detected rate  (h⁻¹) -->
      <TI>8760</TI>              <!-- test interval (h) -->
      <MTTR>8</MTTR>             <!-- mean time to restore (h) -->
      <beta>5</beta>             <!-- CCF factor β (%) -->
      <betad>5</betad>           <!-- CCF factor βd (%) -->
      <M>1</M>                  <!-- votes required -->
      <N>2</N>                  <!-- total channels -->
      <partialTest>false</partialTest>  <!-- true | false -->
      <MT>87600</MT>             <!-- manual test interval (h) -->
      <ptc>0</ptc>              <!-- partial test coverage (%) -->
    </component>
  </category>

  <category type="logic"> <!-- ... --> </category>
  <category type="final_element"> <!-- ... --> </category>

</pfd_library>

Each maps to the corresponding calculator panel automatically.

hosting options

GitHub raw file URL, GitHub Pages, any HTTPS static server. CORS must allow the page origin. For local use, serve with python -m http.server.

sample library

The bundled pfd_library.xml includes 15 components — 5 Sensors, 5 Logic Solvers, 5 Final Elements — with real manufacturer and model references.

Technology

How it's built

Pure HTML + CSS + JavaScript. The Python calculation engine is loaded as WebAssembly via Pyodide and runs entirely client-side.

PyPFD
PFD calculation engine
by Rafael Rocha · pip install PyPFD

Implements RBD and IEC 61508-N2595 PFDavg formulas for MooN architectures with and without partial testing.

Pyodide
Python in the browser
v0.23.4 · via CDN

Compiles CPython to WebAssembly. Installs PyPFD via micropip at startup. No Python server required.

html2canvas
DOM-to-canvas renderer
v1.4.1 · cdnjs

Captures the A4 sheet at 3× resolution for PDF export. Form fields are frozen to static spans before capture.

jsPDF
PDF generation
v2.5.1 · cdnjs

Embeds the high-resolution canvas image into an A4 landscape PDF and triggers browser download.
