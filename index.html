Applications

Two tools, one mission

Both applications share the same PyPFD calculation core. Choose the one that fits your workflow — graphical state modelling or rapid structured documentation.

App 01 ⚛
Markov Solver for PFD

A full graphical Markov chain editor with an interactive canvas. Draw states (Normal, Degraded, Failure), connect them with transition rates, and let the Python engine numerically solve the system of differential equations — returning PFDavg and a time-evolution probability chart.

  • Drag-and-drop canvas: create, connect and label states freely
  • Three state types: Normal (green), Degraded (blue), Failure (orange)
  • Symbolic transition matrix — formulas like λ_du * T using named variables
  • Numerical Markov solver via SciPy (solve_ivp), Matplotlib chart output
  • Analytical PFD sidebar: RBD + N2595, MooN voting, partial test
  • Side-by-side comparison of Markov vs. analytical results
  • Pan, zoom, resizable sidebars, collapsible HUD
  • Save/load full project as JSON · English ⇄ Portuguese i18n

App 02 🧮
SIF Triple Calculator

A structured A4-landscape document for analytical PFD calculations. Three side-by-side panels — Sensor, Logic Solver, Final Element — each computing PFDavg independently. Results combine in a SIF Summary bar with system SIL classification and a contribution chart.

  • Three independent subsystem panels: Sensor · Logic · Final Element
  • Identification fields: Name, Manufacturer, Model, Description
  • Inputs: λdu, λdd, TI, MTTR, β, βd, M, N, partial test (MT + PTC)
  • Dual method output: RBD and IEC 61508-N2595 (PETROBRAS N2595)
  • System PFDavg (series), automatic SIL 1–4 classification
  • Segmented contribution bar chart per subsystem
  • XML component library — load from any URL, auto-filtered by type
  • Export to high-resolution PDF · Save/load JSON project state

🐍

Python in the Browser

Both apps run CPython via Pyodide (WebAssembly). PyPFD is installed at startup via micropip. No Python installation, no server, no build step.

📐

Standards-Based Formulas

RBD method per IEC 61508. Tabular N2595 method per ISA-TR84.00.02 / PETROBRAS N2595. Dual-method output for cross-verification.

📦

Zero-Dependency Deploy

Open the HTML file in any modern browser. Deploy to GitHub Pages with no configuration. Offline use possible after first load caches Pyodide.

Component Library

Shared XML library format

The SIF Calculator loads a component library from any HTTPS URL. Host the XML on GitHub raw files, GitHub Pages, or any static server. Dropdowns auto-filter by subsystem type (sensor / logic / final_element).

pfd_library.xml 
<pfd_library version="2.0">

  <category type="sensor">
    <component id="S01">
      <name>Pressure Transmitter (1oo2)</name>
      <manufacturer>Emerson</manufacturer>
      <model>Rosemount 3051S</model>
      <description>Redundant PT, 1oo2</description>
      <ldu>1.0e-6</ldu>   <!-- h⁻¹ -->
      <ldd>2.5e-5</ldd>   <!-- h⁻¹ -->
      <TI>8760</TI>      <!-- test interval (h) -->
      <MTTR>8</MTTR>      <!-- h -->
      <beta>5</beta>      <!-- CCF β (%) -->
      <betad>5</betad>    <!-- CCF βd (%) -->
      <M>1</M>  <N>2</N>
      <partialTest>false</partialTest>
      <MT>87600</MT>  <ptc>0</ptc>
    </component>
  </category>

  <category type="logic"> <!-- ... --> </category>
  <category type="final_element"> <!-- ... --> </category>

</pfd_library>

sensor
logic

Each maps to the corresponding calculator panel. Dropdowns are pre-filtered — sensors only appear in Calculation 1, etc.

Bundled sample library

pfd_library.xml includes 15 pre-configured components — 5 per category — with real failure rate data and manufacturer references (Emerson, Siemens, Triconex, Rotork, ABB, Metso…).

Hosting

Any HTTPS static server. For GitHub Pages, use the raw file URL. For local use, serve with:

python -m http.server 8080
Technology

Built on open-source

Pure HTML + CSS + JavaScript. The Python engine runs as WebAssembly in the browser — no server, no framework, no build step.

PyPFD
PFD calculation engine
pip install PyPFD · by Rafael Rocha

RBD and N2595 formulas for MooN architectures. Supports full proof tests and partial tests.

Pyodide
Python in WebAssembly
v0.23.4 · CDN

Runs CPython in the browser. PyPFD + SciPy + Matplotlib installed at startup via micropip.

html2canvas
DOM-to-canvas capture
v1.4.1 · cdnjs

Captures the A4 sheet at 3× resolution for PDF export. Form fields are frozen to static spans before capture.

jsPDF
PDF generation
v2.5.1 · cdnjs

Embeds the canvas image into an A4 landscape PDF and triggers browser download.
